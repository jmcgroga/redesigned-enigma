Mathematical Foundations

This chapter introduces several key mathematical concepts used throughout this document. We begin with classical algebra, move to calculus, and conclude with an introduction to linear algebra notation.

1.1 Quadratic Equations

A quadratic equation has the general form ax2 + bx + c = 0 where a, b, and c are real coefficients with a≠0 . The solutions are given by the quadratic formula:

x = −b ± b2 − 4ac 2a

The term inside the square root, Δ = b2 − 4ac , is called the discriminant. When Δ>0 there are two distinct real roots; when Δ=0 there is exactly one repeated real root; and when Δ<0 the roots are complex conjugates.

1.2 Euler's Identity

Widely regarded as the most beautiful equation in mathematics, Euler's identity unifies five fundamental constants:

e iπ + 1 = 0

This follows from Euler's formula e iθ = cosθ + isinθ by substituting θ=π .

1.3 Summation and Integration

The sum of the first n natural numbers is:

∑ k=1 n k = n(n+1) 2

In the continuous case, integration gives us area under a curve. The definite integral of f(x) from a to b is:

∫ a b f(x)   dx = F(b) − F(a)

where F is any antiderivative of f. This result is the Fundamental Theorem of Calculus.

1.4 Mass-Energy Equivalence

Einstein's famous relation between mass and energy:

E = m c2

Here E is energy in joules, m is rest mass in kilograms, and c is the speed of light in a vacuum ( 2.998×108  m/s ).

1.5 Normal Distribution

The probability density function of the normal distribution is:

f(x) = 1 σ 2π e − (x−μ) 2 2σ2

where μ is the mean and σ is the standard deviation. The distribution is symmetric around μ and approximately 68% of values fall within one standard deviation of the mean.

Algorithms and Complexity

This chapter surveys fundamental algorithms — binary search, merge sort, and dynamic programming — and summarises their time and space complexities.

3.1 Binary Search

Binary search finds a target value in a sorted array in O(logn) time by repeatedly halving the search interval. 

def binary_search(arr: list, target: int) -> int:
    """Return index of target in sorted arr, or -1 if not found."""
    low, high = 0, len(arr) - 1

    while low <= high:
        mid = (low + high) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            low = mid + 1
        else:
            high = mid - 1

    return -1

3.2 Merge Sort

Merge sort is a divide-and-conquer algorithm with guaranteed O(nlogn) time complexity. It splits the array in half, recursively sorts each half, then merges the two sorted halves. 

def merge_sort(arr: list) -> list:
    if len(arr) <= 1:
        return arr

    mid = len(arr) // 2
    left  = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return _merge(left, right)


def _merge(left: list, right: list) -> list:
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

3.3 Dynamic Programming — Longest Common Subsequence

The Longest Common Subsequence (LCS) problem finds the longest subsequence common to two sequences. The recurrence is:

L[i][j] = { L[i−1][j−1]+1 if Xi=Yj max(L[i−1][j],L[i][j−1]) otherwise 
def lcs_length(X: str, Y: str) -> int:
    m, n = len(X), len(Y)
    # Build (m+1) x (n+1) table, initialised to zero
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i - 1] == Y[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]

3.5 Fibonacci — Naive vs. Memoised

The naive recursive Fibonacci has exponential complexity O(2n) because it recomputes subproblems. Memoisation reduces this to O(n) . 

from functools import lru_cache

# Naive — O(2^n)
def fib_naive(n: int) -> int:
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)

# Memoised — O(n) time, O(n) space
@lru_cache(maxsize=None)
def fib_memo(n: int) -> int:
    if n <= 1:
        return n
    return fib_memo(n - 1) + fib_memo(n - 2)

# Bottom-up DP — O(n) time, O(1) space
def fib_dp(n: int) -> int:
    if n <= 1:
        return n
    a, b = 0, 1
    for _ in range(2, n + 1):
        a, b = b, a + b
    return b
